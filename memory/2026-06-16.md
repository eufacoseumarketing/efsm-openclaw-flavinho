## Screenshot API — Limites (Testados na EFSM 01)

### Relay (rápido, ~350ms)
- Funciona: scale até 40 + quality até 60
- Funciona: scale 25 + quality 100
- Limite: payload base64 > ~80KB → timeout e fallback pra powershell

### Powershell (lento, ~18s)
- Funciona até o extremo: scale 100 + quality 100 (1MB base64)
- Sempre funciona como fallback quando relay falha
- Scale > 55% NÃO trava (informação anterior estava incorreta)

### Recomendação
- Default: scale=35, quality=30 (relay, rápido, ~40KB, Gemini detecta 15+ elementos)
- Precisão: scale=75, quality=60 (powershell, quando precisar ler texto fino)
- Máximo: scale=100, quality=100 (powershell, ~1MB, só em último caso)

## Lições Técnicas

### Escaping de JSON nos comandos
- ⚠️ NUNCA usar echo com heredoc pra JSON com backslashes — escaping fica errado
- ✅ SEMPRE usar a ferramenta write do OpenClaw pra criar arquivos de payload JSON
- ✅ Depois usar `curl -d @/tmp/pcr_payload.json` pra enviar
- O comando `write` do shell não existe — cuidado pra não confundir com a tool write

### Gemini JSON truncation
- Continua truncando JSON mesmo com maxOutputTokens=8192
- Acontece mais com telas complexas (muitos elementos)
- O recovery script em screenshot_process.py fecha chaves/colchetes mas pode gerar JSON inválido
- Telas simples (UAC, poucos elementos) sempre funcionam bem
- Investigar: aumentar mais os tokens? Prompt mais conciso pedindo menos elementos?

### BITS Download
- `bitsadmin /transfer` é SÍNCRONO e trava o agente em downloads grandes (>100MB)
- `Start-BitsTransfer -Asynchronous` é assíncrono e não trava
- Para usar assíncrono: disparar com `cmd /c start powershell -Command "Start-BitsTransfer ..."`
- Depois verificar com `Get-BitsTransfer` (job sobrevive entre comandos)

### Fluxo de atendimento na prática
- A KB tem o procedimento, mas a execução real encontra obstáculos (UAC, timing, janelas em background)
- CLI primeiro, tela depois — mas quando precisa de GUI, o UAC é o calcanhar de aquiles
- Importante: sempre verificar com screenshot depois de cada ação crítica (UAC, abertura de instalador)
- Não mandar Enters sequenciais sem confirmar que a janela alvo está em foco
