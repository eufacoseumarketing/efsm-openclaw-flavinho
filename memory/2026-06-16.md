## Lições Técnicas

### Escaping de JSON nos comandos
- ⚠️ NUNCA usar echo com heredoc pra JSON com backslashes — escaping fica errado
- ✅ SEMPRE usar a ferramenta write do OpenClaw pra criar arquivos de payload JSON
- ✅ Depois usar `curl -d @/tmp/pcr_payload.json` pra enviar
- O comando `write` do shell não existe — cuidado pra não confundir com a tool write

### Gemini JSON truncation
- Continua truncando JSON mesmo com maxOutputTokens=8192
- Acontece mais com telas complexas (muitos elementos)
- O recovery script em screenshot_process.py fecha chaves/colchetes mas pode gerar JSON inválido
- Telas simples (UAC, poucos elementos) sempre funcionam bem
- Investigar: aumentar mais os tokens? Prompt mais conciso pedindo menos elementos?

### BITS Download
- `bitsadmin /transfer` é SÍNCRONO e trava o agente em downloads grandes (>100MB)
- `Start-BitsTransfer -Asynchronous` é assíncrono e não trava
- Para usar assíncrono: disparar com `cmd /c start powershell -Command "Start-BitsTransfer ..."`
- Depois verificar com `Get-BitsTransfer` (job sobrevive entre comandos)

### Fluxo de atendimento na prática
- A KB tem o procedimento, mas a execução real encontra obstáculos (UAC, timing, janelas em background)
- CLI primeiro, tela depois — mas quando precisa de GUI, o UAC é o calcanhar de aquiles
- Importante: sempre verificar com screenshot depois de cada ação crítica (UAC, abertura de instalador)
- Não mandar Enters sequenciais sem confirmar que a janela alvo está em foco

### UIPI — O bloqueio real (Ananias confirmou)
- **UIPI (User Interface Privilege Isolation)**: Windows bloqueia input injection entre processos de níveis de privilégio diferentes.
- Agent roda como SYSTEM, apps do usuário rodam como `efsm` — UIPI bloqueia cliques e teclas em janelas de apps.
- **Cliques por coordenada funcionam em**: barra de tarefas, menu Iniciar, relógio, área de trabalho, ícones.
- **Cliques por coordenada NÃO funcionam em**: instaladores, janelas de aplicativos (UIPI bloqueia `SendInput` em janelas de outro integrity level).
- **Tab+Enter VIA KVM funciona** (Ananias confirmou): o KVM injeta teclado como dispositivo virtual, contornando UIPI. Mas precisa de navegação por Tab nas janelas.

### SetForegroundWindow — DESCOBERTA CRÍTICA
- KVM keyboard (Tab, Enter) SÓ funciona em janelas de app se a janela tiver foreground focus
- SetForegroundWindow via PowerShell resolve isso:
  ```powershell
  Add-Type -MemberDefinition '[DllImport("user32.dll")]public static extern bool SetForegroundWindow(IntPtr h);'
  $p = Get-Process -Name NomeDoProcesso | Select -First 1
  [Console.Window]::SetForegroundWindow($p.MainWindowHandle)
  ```
- SetWindowPos com HWND_TOPMOST (-1) força a janela ACIMA de tudo (inclusive widgets)
  ```powershell
  Add-Type -MemberDefinition '[DllImport("user32.dll")]public static extern bool SetWindowPos(IntPtr h,IntPtr a,int x,int y,int cx,int cy,uint f);'
  [Win]::SetWindowPos($h, -1, 0, 0, 0, 0, 3)  # HWND_TOPMOST, SWP_NOSIZE|SWP_NOMOVE
  ```

### Windows 11 Widgets — PRATICAMENTE IMATÁVEL
- Widgets panel resistiu a TODOS os métodos de fechamento:
  - Esc (KVM): não fechou
  - Win+W (KVM): não fechou
  - Win+D (KVM): não fechou
  - Clique fora (mouse_event): não fechou
  - Kill do processo Widgets.exe + WebExperience: não fechou
  - FindWindow + ShowWindow(SW_HIDE): não achou a janela (classe Windows.UI.Core.CoreWindow não bate)
- **Única solução**: forçar a janela do instalador PRA FRENTE com SetWindowPos(HWND_TOPMOST)
- Widgets ficou ATRÁS do instalador quando ele foi promovido a topmost

### JSON escaping — REGRA REAFIRMADA
- ⛔ NUNCA usar echo/heredoc no shell pra criar JSON com backslashes (caminhos Windows)
- ✅ SEMPRE usar a ferramenta `write` do OpenClaw pra criar arquivos de payload
